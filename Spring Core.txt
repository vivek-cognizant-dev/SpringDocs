12. LifeCycle of Spring Bean
	1. Initially spring framework will read the instructions from the xml file and then instantiate the bean.
	2. It calls the constructor to perform some action ate the time creating the object.
	3. It initialize properties of all the beans.
	4. Then it will call the post initialization method.
	5. Now the bean is ready to use.
	6. Beans gets destroyed and destroy method will be called.

	to destroy we use - cnt.registerShutdownHook();

13. Autowiring  : Autowiring is a feature where spring automatically looks for the dependency and inject the object in the current class
	three types of autowiring
	1. By name 
		<bean id="t1" class="com.cts.Triangle" autowire="byName">
			<property name="name" value="obtuse"></property>
		</bean>
	2. By Type	
		<bean id="t1" class="com.cts.Triangle" autowire="byType">
			<property name="name" value="obtuse"></property>
		</bean>
	3. By Constructor	
		<bean id="t1" class="com.cts.Triangle" autowire="constructor">
			<constructor-arg index="0" value="obtuse"></constructor-arg>
		</bean>
